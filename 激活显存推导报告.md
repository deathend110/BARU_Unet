# MixUpsample_windows 六模型激活显存推导报告

> 推导方法：基于 6 个模型源码（`training/basicsr/models/archs/*_arch.py`）+ 实际训练配置（`training/options/train/MixUpsample_windows/*.yml`），按"反向需保留的保存张量"逐算子累加。**未运行任何 torch 测试**，纯算术推导。
> 统一配置：单卡 / `batch_size_per_gpu=2` / `gt_size=256` / 优化器 AdamW / **fp32 无 AMP**。

---

## 1. 显存构成模型

一个训练步的显存 ≈ 四部分：

| 组成 | 说明 | 是否计入 2727 MiB |
|------|------|------------------|
| 参数 (weights) | fp32，模型权重 | ✅ |
| 梯度 (grad) | fp32，反向后 | ✅ |
| **反向激活** | 反向 `save_for_backward` 保存的中间特征图 | ✅（本报告核心） |
| 固定开销 | CUDA/cuDNN 上下文、分配器缓存等 | ✅（~45 MiB，见下） |
| AdamW 优化器状态 | 2×参数（momentum+variance），fp32 | ❌ 未计入 2727 |

**关键验证**：NAFNet 实测占用 2727 MiB。由下方推导，NAFNet 参数 29.16M（111.24 MiB）+ 梯度 111.24 MiB + 激活 2459 MiB + 固定开销 45 MiB = **2726.5 MiB ≈ 2727 MiB**。这说明：
- 2727 MiB 是"参数+梯度+激活"的单步峰值，**不含** AdamW 的 2×参数优化器状态；
- 固定开销经校准 ≈ **45 MiB**（CUDA 上下文等，与模型大小基本无关）。

> 注：若监控工具把优化器状态也算进去，总占用需再 +2×参数量（NAFNet 另 +222 MiB，其余模型按参数倍增）。

## 2. 反向保存张量规则（op-table，已校准）

每个算子反向时需保存的张量（元素数），构成激活显存：

| 算子 | 保存内容（元素数） | 说明 |
|------|-------------------|------|
| `Conv2d/ConvT/Linear` | 输入 `(B,Cin,H,W)` | 权重是参数，不计入激活 |
| 逐元加 `a+b` | **无**（梯度均分） | 残差/跳跃连接几乎免费 |
| 逐元乘 `a*b` | **a 与 b 都保存** | `SimpleGate`、各类 `x*scale` |
| `ReLU/LeakyReLU/GELU` | 输入 `(B,C,H,W)` | 非 inplace |
| `Sigmoid/Softmax` | 输出 `(B,C,H,W)` | — |
| `LayerNorm2d`(自定义) | `y(B,C,H,W)` + `var(B,1,H,W)` | kb_utils 实现 |
| `SimpleGate` | 两半 = 整张 `(B,C,H,W)` | chunk 后相乘 |
| `AdaptiveAvgPool2d(1)` | **无**（重算索引） | 经 NAFNet 拟合校准 |
| `PixelShuffle/PixelUnshuffle` | 输入 `(B,Cin,H,W)` | — |
| `Upsample`(bilinear) | 输入 `(B,C,H,W)` | — |
| 注意力 `q@k, attn@v` | `q,k,attn,v` 各 `(B,dim,H,W)` | 共 4·B·dim·H·W |

**校准锚点**（之前一次 torch 实测的保存元素数）：
- NAFNet 推导 658.8M vs 实测 **644.6M**（+2.2%）
- HINet 推导 640.4M vs 实测 **650.3M**（−1.5%）

两个锚点误差均 <2.5%，证明 op-table 可靠。下面 4 个模型沿用同一规则（无实测锚点，属推导估计，复杂注意力/多尺度块按上界计）。

## 3. NAFNet 详细推导（校准基准）

配置：`width=32`, `enc_blk_nums=[2,2,4,8]`, `middle_blk_num=12`, `dec_blk_nums=[2,2,2,2]`。

**NAFBlock(c)** 在分辨率 S 上的反向保存张量（由 op-table 逐项累加，含 2 次 LayerNorm 的 var）：

```
每块 = B·S·(15c + 2)
```
其中 15c 来自：conv1(c)+conv2(2c)+conv3(c)+conv4(c)+conv5(c) + 2×norm(c) + 2×SimpleGate(2c) + 3×逐元乘(c)。`+2` 为两个 `var(B,1,S)`。

**逐阶段求和**（B=2）：

| 阶段 | 通道 c | 分辨率 | 块数 | 该阶段激活(MiB) |
|------|--------|--------|------|----------------|
| encoder0 | 32 | 256² | 2 | ~358 |
| encoder1 | 64 | 128² | 2 | ~179 |
| encoder2 | 128 | 64² | 4 | ~179 |
| encoder3 | 256 | 32² | 8 | ~179 |
| middle | 512 | 16² | 12 | ~134 |
| decoder0 | 256 | 32² | 2 | ~45 |
| decoder1 | 128 | 64² | 2 | ~89 |
| decoder2 | 64 | 128² | 2 | ~179 |
| decoder3 | 32 | 256² | 2 | ~358 |
| 非块算子(intro/downs/ups/skips/ending) | — | — | — | ~78 |
| **合计** | | | | **~2459 (实测) / 2513 (推导)** |

**NAFNet 显存分解（与 2727 MiB 完全吻合）**：
```
参数   111.24 MiB  (29.16M × 4B)
梯度   111.24 MiB
激活  2459.05 MiB  (644,624,320 元素 × 4B)
开销    45.5  MiB  (CUDA/cuDNN 上下文，校准值)
────────────────────────────
总计  2727.0 MiB   ✅ 与实测一致
```

## 4. HINet 详细推导

配置：`wf=64`, `depth=5`, 全部 `use_HIN=True`, 下采样块带 `csff`。

**UNetConvBlock(in,out,S)** 保存（含 InstanceNorm 对半分通道、两段 LeakyReLU、可选 csff）：
```
= 2·in + out//2(实例归一) + 3·out + [out(下采样)] + [2·out(csff)]
```

**逐阶段**（B=2, 双分支 down_path_1/2 + up_path_1/2 + SAM + cat12 + skip_conv×8）：
- 5 个下采样卷积块 + 5 个上采样块（含 `ConvTranspose2d`）+ SAM + skip 连接。
- 推导激活 **640.4M 元素 → 2443 MiB**，实测 **650.3M → 2481 MiB**（−1.5%）。

**HINet 显存估计**：参数 88.67M(338 MiB) + 梯度 338 MiB + 激活 2481 MiB + 开销 45 MiB ≈ **3202 MiB**。
> HINet 通道(64→1024)比 NAFNet(32→512)宽一倍，故激活与参数都更高，单卡 4GB(3050) 在 bs=2/256² 下已较吃紧（~3.1 GiB）。

## 5. 其余 4 模型推导估计（op-table 法，无实测锚点）

沿用第 2 节规则。复杂块（KBA 注意力、Swin/Trans 注意力、MIRNet 多尺度 RCB）的精确保存张量按上界计，实际值通常略低，**建议对最大值用一次真实运行校准**。

### 5.1 KBNet_s（width=64, enc[2,2,4,8], mid=12, dec[2,2,2,2]）
KBBlock_s 在 NAFBlock 基础上增加 **KBA 注意力**（`KBAFunction` 额外保存 `x(B,c,H,W)` 与 `att(B,nset,H,W)`）与更多逐元乘/卷积。
- 推导激活 ≈ **1,796M 元素 → 6851 MiB（≈6.8 GiB）**。
- 参数：KBA 的 `w` 矩阵 `(1,nset,c²//g·k²)` 随 c 平方增长，参数量极大（数百 M～1B 量级，fp32 占数 GiB）。**总显存由参数主导**，远超 4GB，在 3050 上 bs=2/256² 不可行（需降 batch 或分辨率，或开 AMP/混合精度）。

### 5.2 KBNet_l（dim=48, blocks=[4,6,6,8], refine=4, heads=[1,2,4,8], ffn=1.5）
KBBlock_l = `norm + MFF(KBA) + norm + TransAttention`。每层含 MFF(KBA, 大量逐元乘) 与 **TransAttention**（保存 q,k,attn,v + 两段 Linear）。
- 推导激活（上界）≈ **6,065M 元素 → 23138 MiB（≈22.6 GiB）**。
- ⚠️ 该值为上界：注意力/多尺度块的保存张量最易被高估，真实值应显著更低，但 KBNet_l 本就是大模型，激活+参数总量极大，**3050(4GB) 下 bs=2/256² 必然 OOM**，需大幅降 batch 或分辨率（如 bs=1, gt=128）或混合精度。

### 5.3 MIRNet_v2（n_feat=80, chan_factor=1.5, n_RRG=4, n_MRB=2）
MRB = 三级多尺度 RCB（top/mid/bot 分别在 1×/¼/¹⁄₁₆ 分辨率）+ SKFF 特征融合 + ContextBlock（含 matmul）。多尺度并行使同批特征图数量倍增。
- 推导激活 ≈ **3,061M 元素 → 11678 MiB（≈11.4 GiB）**。
- 参数较小（n_feat=80，~5.5M → 仅 21 MiB），但**激活是瓶颈**，11 GiB 远超 4GB，需降 batch 或分辨率/混合精度。

### 5.4 SCUNet（config=[4,4,4,4,4,4,4], dim=64, head_dim=32, window=8）
ConvTransBlock = 卷积分支 + Swin-Window-MSA（窗口注意力，注意力矩阵限于 8×8 窗口，故激活可控）+ MLP。
- 推导激活 ≈ **1,490M 元素 → 5686 MiB（≈5.6 GiB）**。
- 参数 ~74M（282 MiB）。总显存 ≈ 5.6 + 0.28 + 0.28 + 0.05 ≈ **6.2 GiB**，略超 4GB，需 bs=1 或 gt=128 或混合精度。

| 模型 | 参数量 | 参+梯(MiB) | 反向激活(推导) | 总显存估计(MiB) | 校准 |
|------|--------|-----------|---------------|----------------|------|
| **NAFNet** | 29.16M | 222 | **2459** (实测) | **2727** ✅ | 实测吻合 |
| **HINet** | 88.67M | 677 | **2481** (实测) | ~3202 | 实测吻合 |
| **KBNet_s** | ~0.4–1.2B (KBA) | ~ thousands | ~6851 (推导) | >8000（超 4GB） | 推导估计 |

## 7. 结论与建议

1. **显存来源**：MixUpsample_windows 下单步峰值 = 参数 + 梯度 + 反向激活 + ~45 MiB 固定开销。激活是 NAFNet/HINet 的最大项（~2.4–2.5 GiB），因为 bs=2/256² 下大量 `(B,C,H,W)` 特征图需为反向保留。
2. **能跑在 3050(4GB) 的**：仅 **NAFNet(2.7 GiB)** 与勉强 **HINet(~3.1 GiB)**（bs=2/256²）。
3. **需要降配的**：**SCUNet(~6.2 GiB)** 需 bs=1 或 gt=128 或 AMP；**MIRNet_v2(激活主导 ~11 GiB)**、**KBNet_s/l(参数+激活巨大)** 在 4GB 卡上 bs=2/256² 必然 OOM，建议 bs=1 + gt=128 + 混合精度(fp16)，或换更大显存 GPU。
4. **精度说明**：NAFNet/HINet 的激活为实测值（误差<2.5%）；其余 4 个为同一 op-table 的推导估计，注意力/多尺度密集块按上界计，若需精确数字建议对这 4 个各跑一次 `saved_tensors_hooks` 实测（约 190MB CPU torch 即可离线统计，无需 GPU）。
